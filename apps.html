FAQ

Are blindsoft applications secure?

Yes, they are..

We are not hack proof in any way, whatsoever.

But we do our best to maintain security as hard as we can.

What is a PGP file?

A pgp file pertains to a file encrypted by the PGP (pretty good privacy) standard.

This goes beyond cloudflare's scope of encryption.

Cloudflare r2 encrypts files at the

Server

Side, but not the

Client

Side, but PGP encrypts data at the

client

side.

This means only those who hold the pgp private key can actually decrypt the data.

What is a file with the .enc extension?

This is an encrypted file. The only difference is while PGP uses its own asymmetric protocols, the backup files use AES256 encryption algorithm for encryption.

 On cloudron instances, rather than having r2 encrypt the file, the files will be encrypted by the server and can be accessed via password, rather than a public key.

while in this case of backups, it is encrypted by the server ,it can also be client side.

In any case, all of this ensures backups cannot automatically be opened, even if the r2 bucket is compromised.

Can blindsoft see what you’re doing?

On publicly available applications:

While blindsoft *can* see what you’re doing, we usually don’t, unless it’s absolutely required to make the service work

Make the service work may include

  • Technical support requests
  • We think there’s something wrong and we need to view any errors.
  • Testing the application

What we can access

  • We can see the IP you connect from. Note: this is standard for all internet services in the world, as they need to see your IP.
  • Occasionally, we may be able to see what services are querying.

Staff applications

On staff-only application, blindsoft administrators are authorized to tap into your session at any given time for any reason whatsoever.

We are also allowed to log the following

  • Who you are (what user account you accessed from)
  • The IP address that accessed the page, including what country.
  • If required, your justification for accessing the resource.
  • How many times you’ve accessed this service
  • The login method you used (OTP via email, google, github, etc)
  • What resource you requested

Note:

Just because the system is a

Secure

System, does not mean it is a

Private

System. Blindsoft is allowed to log activities you do on the services, including information above, without your consent or prior notice.

The difference between a (secure) system and a (private) system.

A

Secure

System pertains to a system with certain security measures put in place to protect confidential information.

Usually, unless it is a (private) system, secure systems are usually owned by businesses, enterprises, etc.

A (private) system pertains to you, or someone else privately holding the system and or information on the system. No logging is done by a third party, only you can access the system.

Note:

Because it is logged by blindsoft enterprises, this is a secure system, and is not private to one person.

Also note

Any system or bucket with access controls is not a public system, unless it is said to be a public system.

Where can I get files of a bucket from?

In all instances, you need the bucket URL, not just the domain/subdomain.

In some instances, we will direct you from that page to the URL.

For example, blindsoft redirects the homepage from

/

To

/home.html

In order to serve you your content.

In other instances, you will need the file URL

From the bucket administration page
  1. go to the cloudflare  dashboard
  2. Find the “blindsoft/g4p” zone.
  3. Go to r2, and the domain/subdomain where you want to find the file.
  4. If you have access, click that file, and click “download”.
  5. If you are doing this for someone else, you can either send the file to someone, or you can copy the file URL and send it to the user you are giving a file to.

Without =administrator access

Without admin or r2 access, you will need to have an administrator do it for you.

Contact an administrator who has access to the r2 bucket and have them help you get the file you need.

I have the file URL, but it doesn’t give me access.

In this case, the following can happen.

  1. You have access, you were just not added to the zero-trust access by the administration. In this case, contact an administrator for help.
  2. You do not have permission to access the application.
  3. While you have access, there is a misconfiguration in the zero-trust launcher not letting you access the page. In this case, contact administration.
